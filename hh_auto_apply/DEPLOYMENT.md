# Развертывание HH Auto Apply с веб-интерфейсом

## Обзор

HH Auto Apply — это инструмент для автоматического отклика на вакансии на hh.ru. Проект включает:
1. **Ядро автоотклика** — Python-скрипты с Selenium для автоматизации браузера
2. **Веб-интерфейс** — FastAPI + Jinja2 + Bootstrap для управления через браузер

## Архитектура

```
hh_auto_apply/
├── src/                    # Исходный код ядра
├── web/                   # Веб-интерфейс (FastAPI)
│   ├── app.py            # Основное приложение
│   ├── core_integration.py # Адаптер для интеграции с ядром
│   ├── job_manager.py    # Менеджер асинхронных задач
│   ├── templates/        # Jinja2 шаблоны
│   └── static/           # CSS, JavaScript
├── config/               # Конфигурационные файлы
├── data/                 # База данных, история, шаблоны писем
└── Dockerfile           # Многостадийный Docker образ
```

## Способы развертывания

### 1. Локальная разработка (без Docker)

#### Требования
- Python 3.12+
- Chromium/Chrome браузер
- Установленные системные зависимости (libgtk, libnss3 и т.д.)

#### Установка
```bash
cd hh_auto_apply

# Установка зависимостей ядра
pip install -r requirements.txt

# Установка зависимостей веб-интерфейса
pip install -r web/requirements.txt
```

#### Запуск веб-интерфейса
```bash
cd web
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

Откройте браузер по адресу: http://localhost:8000

#### Запуск ядра автоотклика (CLI)
```bash
python scheduler.py --interval 3600  # интервал в секундах
```

### 2. Развертывание с Docker (рекомендуется)

#### Сборка образов
```bash
# Сборка всех образов
docker compose build

# Или сборка конкретного сервиса
docker compose build web
docker compose build scheduler
```

#### Запуск сервисов
```bash
# Запуск веб-интерфейса
docker compose up web

# Запуск планировщика откликов
docker compose up scheduler

# Запуск всех сервисов
docker compose up
```

#### Профили Docker Compose
- `web` — веб-интерфейс на порту 8000
- `scheduler` — планировщик автооткликов
- `test` — запуск тестов

### 3. Продакшн развертывание

#### Настройка конфигурации
1. Скопируйте пример конфигурации:
   ```bash
   cp config/example_profile.json config/my_config.json
   ```
2. Настройте файлы:
   - `config/accounts.json` — учетные записи hh.ru
   - `config/my_config.json` — общие настройки
   - `data/templates/` — шаблоны сопроводительных писем

#### Переменные окружения
Создайте файл `.env` в корне проекта:
```env
HH_TEST_KEYWORDS=Python
HH_TEST_AREA=1
WEB_HOST=0.0.0.0
WEB_PORT=8000
```

#### Запуск в фоновом режиме
```bash
docker compose up -d web scheduler
```

#### Мониторинг логов
```bash
docker compose logs -f web
docker compose logs -f scheduler
```

## Конфигурация веб-интерфейса

### Управление аккаунтами
1. Перейдите на страницу "Аккаунты" (http://localhost:8000/accounts)
2. Добавьте учетные записи hh.ru с логином и паролем
3. Настройте фильтры поиска для каждого аккаунта

### Настройка фильтров поиска
- Ключевые слова
- Регион (area)
- Опыт работы
- Тип занятости
- Зарплата

### Шаблоны сопроводительных писем
- Поддерживаются переменные: `{company_name}`, `{position}`, `{vacancy_id}`
- Шаблоны хранятся в `data/templates/`
- Редактирование через веб-интерфейс

### Запуск задач откликов
1. На странице "Задачи" создайте новую задачу
2. Выберите аккаунты и режим работы:
   - `auto` — автоматический поиск и отклик
   - `manual` — только отклик на вакансии из manual_vacancies.txt
   - `recommendations` — рекомендации hh.ru по резюме
3. Мониторинг прогресса через WebSocket

## Режим рекомендаций hh.ru

Для использования рекомендаций по резюме:
1. Получите ID резюме из URL hh.ru
2. Используйте URL: `https://hh.ru/search/vacancy?resume={resume_id}`
3. В веб-интерфейсе выберите режим "recommendations" и укажите resume_id

## Безопасность

### Хранение паролей
- Пароли хранятся в `config/accounts.json` в незашифрованном виде
- **Рекомендация для продакшна**: Используйте секреты Docker или внешнее хранилище

### Доступ к веб-интерфейсу
- По умолчанию веб-интерфейс доступен без аутентификации
- Для ограничения доступа настройте reverse proxy с аутентификацией

### Сетевая изоляция
- Веб-интерфейс слушает на `0.0.0.0:8000`
- Используйте firewall для ограничения доступа к порту

## Устранение неполадок

### Проблема: Ошибка "Chromium not found"
**Решение:** Убедитесь, что Chromium установлен в контейнере. Проверьте Dockerfile.

### Проблема: Веб-интерфейс не загружает шаблоны
**Решение:** Проверьте права доступа к папке `web/templates/`. Убедитесь, что файлы скопированы в контейнер.

### Проблема: Ошибка аутентификации на hh.ru
**Решение:**
1. Проверьте логин и пароль в `accounts.json`
2. Убедитесь, что аккаунт не заблокирован
3. Попробуйте выполнить вход вручную через браузер

### Проблема: WebSocket соединение разрывается
**Решение:** Проверьте настройки прокси и firewall. Убедитесь, что порт 8000 открыт.

## Масштабирование

### Горизонтальное масштабирование
- Каждый экземпляр веб-интерфейса работает независимо
- Используйте внешнюю базу данных (PostgreSQL) вместо SQLite
- Настройте балансировщик нагрузки для нескольких экземпляров

### Распределенные задачи
- Для распределенного выполнения задач используйте Celery или Redis Queue
- Настройка выходит за рамки текущей документации


3. Пересоберите образы:
   ```bash
   docker compose build --pull
   ```
4. Запустите сервисы:
   ```bash
   docker compose up -d
   ```
